# Design Tokens

## Token 域

- `color`：品牌、语义状态、文本层级、背景层级、边框。
- `typography`：字号、字重、行高、标题层级、数字显示。
- `spacing`：页面边距、区块间距、组件内边距、表单间距。
- `radius`：按钮、卡片、输入框、弹层、标签等圆角层级。
- `shadow`：卡片、浮层、悬停态、聚焦态。
- `motion`：进入、退出、切换、反馈、加载。

## 约束

- 组件消费 token，不直接消费原始视觉常量。
- 新 token 需要说明来源、适用范围和替代不了现有 token 的原因。
- Token 命名应表达语义，不绑定某个页面或业务临时叫法。

## 默认建议

- 正文排版以可读性优先，先保证层级，再追求视觉个性。
- 状态色要和文案、图标或形态搭配，不单靠颜色传递语义。
- Motion token 要服务于状态变化与空间连续性，不做纯装饰噪音。
