# Interaction, Accessibility, and Feedback

## 交互

- 关键动作必须有即时反馈，用户需要知道“已收到、处理中、已完成、失败了什么”。
- 微交互应帮助理解状态变化，不应造成等待感或视线噪音。
- 高风险操作要给出确认、撤销或恢复路径。

## 可访问性

- 默认以键盘可达、焦点可见、标签可理解、对比达标作为底线。
- 所有交互元素都需要足够触达面积，默认不低于 `44x44`。
- 状态变化要能被辅助技术感知，不能只在视觉上轻微变化。

## 表单与导航

- 表单字段要有标签、提示和错误修正路径。
- 导航结构要稳定，返回路径和当前定位要清楚。
- 弹层、抽屉、下拉等临时界面需要处理焦点和关闭行为。
