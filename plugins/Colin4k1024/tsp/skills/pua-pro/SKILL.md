# PUA Pro

## 用途

- 当你不只想解决当前任务，还想把执行风格固化成长期习惯时使用。

## 默认做法

1. 继承 [pua](../pua/SKILL.md) 的全部核心约束。
2. 复用 hooks 写入的 builder journal 和状态文件看趋势，不只看单次成败。
3. 定期回看失败计数、闭环证据密度、是否反复在同类问题上打转。
4. 用 must / should / could 校准“够用”与“过度折腾”的边界。
