# Display Protocol

## 原则

- 简单任务只输出开头和结尾两句旁白，不刷屏。
- 复杂任务可以加 Sprint Banner、阶段状态、KPI 卡，但只在真正有阶段切换时使用。
- 若输出面板，优先使用 Unicode 方框字符，不用 markdown 表格伪装状态面板。

## 推荐面板

```text
┌──────────────────────────────┐
│ PUA Sprint                   │
├──────────────────────────────┤
│ Flavor: 🟠 阿里              │
│ Route: debug -> 华为         │
│ Level: L2                    │
│ Failure Count: 3             │
└──────────────────────────────┘
```

## 推荐用法

- 开工旁白：说明当前味道、路由原因和当前目标。
- 中途旁白：只有当发生额外有价值工作、方法论切换或升级压力时再输出。
- 收尾旁白：明确闭环证据、验证动作和遗留风险。

## 禁止项

- 单行修改也输出大面板。
- 没有验证证据却输出“已完成”的 KPI 卡。
- 用 `[PUA生效 🔥]` 标记本职动作，比如“读了文件”“写了代码”。
