# Methodology Router

## 默认路由

| 任务类型 | 推荐味道 | 方法论 |
| --- | --- | --- |
| Debug / 修 Bug | 🔴 华为 | RCA + 蓝军自攻击 |
| 新功能实现 | ⬛ Musk | question -> delete -> simplify -> accelerate -> automate |
| 架构 / 方案 | 🔶 Amazon | Working Backwards |
| 调研 / 搜索 | ⚫ 百度 | 搜索优先，一切结论先检索 |
| 性能优化 | 🟡 字节 | A/B test + 数据驱动 |
| 部署 / 运维 | 🟠 阿里 | 定目标 -> 追过程 -> 拿结果 |
| 模糊任务 | 🟠 阿里 | 通用闭环 |

## 失败切换链

- 原地打转：Musk -> 拼多多 -> 华为
- 放弃甩锅：Netflix -> 华为 -> Musk
- 质量差：Jobs -> 小米 -> Netflix
- 没搜就猜：百度 -> Amazon -> 字节
- 被动等待：京东 -> 美团 -> 阿里
- 空口完成：字节 -> 京东 -> 阿里

## 切换前三问

1. 当前方法论的核心步骤真的走完了吗？
2. 失败是方法不对，还是执行不到位？
3. 新方法能否真正带来新的信息增量？
